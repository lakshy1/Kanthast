Auth/API Fix Summary

1) OTP pre-save crash (`next is not a function`)
- File: Server/models/OTP.js
- Problem: The Mongoose pre-save hook was `async function(next)` and also calling `next()`. In async middleware this can throw and cause 500 errors.
- Fix: Changed to pure async hook without `next()`.
- Result: `sendotp` no longer returns 500 while email still sends.

2) OTP mismatch during signup (`Invalid or expired OTP`)
- File: Server/controllers/Auth.js
- Problems:
  - Email case/whitespace inconsistencies between sendOTP and signup.
  - Multiple OTP records for same email could cause mismatch with latest OTP.
  - OTP field could include spaces/string mismatch.
- Fixes:
  - Normalize email (`trim().toLowerCase()`) in sendOTP, signup, login.
  - Normalize OTP (`String(otp).trim()`) in signup.
  - In sendOTP, delete older OTPs for that email before creating a new one.
  - In signup, fetch latest OTP for email and compare value.
- Result: Fresh OTP from `sendotp` works reliably in `signup`.

3) Schema/controller mismatches
- Files:
  - Server/models/User.js
  - Server/models/Profile.js
  - Server/models/Tag.js
- Problems:
  - User schema lacked fields used by controllers (`email`, `contactNumber`, reset token fields).
  - Profile schema required fields not used at creation, causing logical mismatch.
  - Tag schema lacked `description` though controller uses it.
- Fixes:
  - Added needed User fields.
  - Simplified Profile to match actual usage.
  - Added `description` in Tag schema.

4) ESM/CommonJS runtime breakages
- Files:
  - Server/models/Section.js
  - Server/models/subSection.js
  - Server/models/ratingAndReview.js
  - Server/models/courseProgress.js
- Problem: Project uses `type: module` but these files used `require/module.exports`.
- Fix: Converted to `import/export default`.

5) Auth hardening and consistency
- File: Server/controllers/Auth.js
- Fixes:
  - Added required-field validation.
  - Added `accountType` guard (`Student`/`Instructor` only at signup).
  - Login now returns sanitized user object (password removed).

6) Course and payment flow fixes
- Files:
  - Server/controllers/Course.js
  - Server/routes/Course.js
  - Server/controllers/Payment.js
  - Server/routes/Payments.js
  - Server/config/razorpay.js
- Fixes:
  - Course creation now uses authenticated user, not `userId` from request body.
  - Instructor-only guard on course creation route.
  - Added missing payment verify route (`/api/v1/payments/verify`).
  - Fixed enrollment duplicate checks using ObjectId-to-string compare.
  - Made Razorpay config safe when keys are missing (no startup crash).

7) Missing imports and route wiring
- Files:
  - Server/controllers/Category.js
  - Server/controllers/RatingsAndReviews.js
  - Server/controllers/SubSection.js
  - Server/routes/User.js
- Fixes:
  - Added missing imports (Category, mongoose).
  - Corrected SubSection model import path/case.
  - Wired change-password/reset-password routes.

How to test quickly now
1. POST /api/v1/auth/sendotp with email
2. POST /api/v1/auth/signup with same email + fresh OTP
3. POST /api/v1/auth/login with credentials
4. Use returned token on protected route (e.g., GET /api/v1/profile)

Notes
- If `sendotp` succeeds but email delays, always use the latest OTP generated.
- OTP TTL is controlled by OTP model expiry (5 minutes).
